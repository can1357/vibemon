# Legacy `ModalClient` compatibility protocol design

> **Status:** design only. `vmon.v1` remains Vibemon's implemented and supported public API. Nothing in this document is currently mounted by `vmon serve`.

This design covers the exact 212-RPC `ModalClient` surface supplied for this work. It treats that surface as an opt-in compatibility boundary over Vibemon's canonical engine, not as a second implementation of scheduling, sandboxes, functions, artifacts, snapshots, credentials, or volumes.

## Compatibility baseline

The supplied method list omits protobuf fields, enum numbers, the package name, metadata, and behavioral rules. A candidate authoritative schema was therefore located and pinned before designing handlers:

- native gRPC method prefix: `/modal.client.ModalClient/`
- [Apache-2.0 upstream license](https://github.com/modal-labs/modal-client/blob/57d76b553013a00b531af1fa69c9e7173ad6c117/LICENSE)

A descriptor comparison found all 212 supplied request types, response types, and streaming cardinalities in that pinned schema with zero signature mismatches. The requested subset has 201 unary RPCs and 11 unary-request/server-streaming RPCs. The candidate schema is necessary but not sufficient. Exact status details, cursor inclusivity, TTLs, list ordering, retry behavior, and old/new field precedence need captured-client fixtures. ## Decisions

1. **Opt-in, native gRPC only.** Add a runtime gate such as `--legacy-modal-protocol`; default off. Mount the service on the existing authenticated HTTP/2 TCP/UDS listener. Do not add REST routes and do not expose it through the browser `/grpc` WebSocket bridge.
2. **Canonical domains stay authoritative.** Legacy handlers translate and call shared application/domain operations. They do not call back through loopback gRPC and do not copy SQL, scheduler, mesh-routing, encryption, or VMM logic from `vmond/src/api/grpc.rs` and the engine.
3. **Exact wire profile, explicit behavioral profile.** Vendor the selected schema under its Apache-2.0 terms, preserve attribution, mark modifications, pin its hash, and build golden descriptors. Gate honest clients by supported `x-modal-client-version`; the header is a compatibility hint, never authorization.
4. **Only the 212-method allowlist is reachable.** Generate or wrap the service from a clearly marked derivative whose `ModalClient` block contains exactly the requested methods. A descriptor test fails on missing, changed, or extra methods. The 20 additional upstream methods are not registered merely because their messages remain in the vendored file.
5. **Fail closed.** A method or request shape without a truthful canonical/new-domain implementation returns `UNIMPLEMENTED`. Required unsupported fields reject the entire request. Empty success, invented counters, guessed tokens, ignored options, and partial writes are forbidden.
6. **Canonical gaps become reusable domains.** Dicts, queues, environment policy, generic secrets, task sessions, and volume file generations are internal application domains independent of the legacy wire types. The adapter remains translation-only.

### Protobuf and route assembly

Add the pinned, attributed schema to the `vmon-proto` crate and generate a separate `modal::client` Rust module from `proto/build.rs`; do not merge its types into `vmon::v1`. The generated service implementation may be mechanically large, but each method should delegate immediately to a domain-specific adapter.

Keep route builders separate. `vmond/src/api/routes.rs` may merge the opt-in legacy native router beside `grpc::routes`, while `bridge_ws` continues to receive only the canonical `grpc::service` routes. Otherwise the current bridge's arbitrary method forwarding would expose the compatibility service unintentionally. Update `is_static_web_path` so legacy gRPC paths never fall through to static content.

Current authorization in `vmond/src/api/state.rs` recognizes `/vmon.v1.*` for gRPC failures and method policy. Add exact recognition for `/modal.client.ModalClient/`; do not use a broad `/modal.` prefix. Replace implicit "unknown path is non-admin" behavior with a static, exhaustive policy table generated or checked against all 212 methods.

### Application boundary

`GrpcApi` currently contains useful translation and some owner-routing/application logic. Extract reusable operations where necessary so canonical and legacy transports call one implementation. The compatibility layer must preserve:

- mesh owner forwarding and principal propagation for sandbox operations;
- immutable function and app revisions plus compare-and-swap activation;
- durable call/input/result sequencing and fenced leases;
- registry lifecycle generations and idempotent sandbox creation;
- artifact digest verification and image realization;
- encrypted credential/volume ownership and attachment locks.

A compatibility handler may compose multiple reads, but a mutation spanning canonical and compatibility state needs one transaction or a recoverable intent/outbox state machine. It may not perform two unrelated commits and report success between them.

## Authentication and authorization

### Credential translation

For TCP, accept the stock client's token pair only through a structured compatibility credential directory that resolves `(token_id, constant-time secret verifier)` to a Vibemon `Principal`. Bearer authentication may remain available for native Vibemon callers. Token IDs do not select a tenant by themselves; client type/version/node/platform headers are untrusted diagnostics.

Resolve credentials before resource lookup and strip/redact `x-modal-token-secret` before tracing, forwarding, or handler metadata inspection. Never persist, echo, place in error details, or forward the secret to mesh peers. UDS retains same-UID/root local-admin authentication. Enabling the legacy TCP route should require configured credentials even if another local route permits anonymous loopback use.

### Exhaustive method policy

Every method belongs to one policy class:

| Class | Rule |
|---|---|
| Tenant | Requires an authenticated tenant principal; every ID, name, cursor, alias, artifact, and list query is tenant-scoped before access. |
| Admin | Environment identity/RBAC, cluster inventory, service-user, or other workspace-wide administration; tenant tokens cannot invoke it. |
| Worker | Attempt, container, task, and worker-side function leasing/output methods require transport authentication plus a server-issued, short-lived capability bound to tenant, task/sandbox, generation, audience, and expiry. `x-modal-client-type` is not proof. |
| Disabled | Unsafe or unsupported methods return `UNIMPLEMENTED` before domain mutation. |

Vibemon currently has only `Admin` and `Client` principals. Worker-only methods remain disabled until a real worker capability issuer/verifier exists. Token-returning calls (`AuthTokenGet`, `CurlGetAuthToken`, connect/router grants) may only mint least-privilege, short-lived capabilities; they must never return the server bearer, tenant token, mesh credential, or canonical tunnel token.

Cross-tenant absent and foreign IDs should generally be indistinguishable as `NOT_FOUND`. Authorization failures occur before expensive decoding/materialization and are audited without secret values or executable payloads.

## Wire behavior shared by all methods

### Idempotency and retries

The pinned Python client retries unary calls on transient statuses using one `x-idempotency-key`. Treat that header as part of the protocol:

- scope a key to `(principal, full method path)`;
- hash the normalized request bytes/semantics;
- commit mutation outcome and replay record atomically;
- replay the original encoded response/status for the same key and hash;
- reject key reuse with a different hash;
- persist replay across process restart for the selected compatibility window.

This includes destructive reads such as queue take/dict pop: the first transaction consumes and stores the exact response, and a transport retry replays it without consuming again. A missing key can use only a proven natural idempotency boundary such as get-or-create name uniqueness. Never derive a key from mutable wall-clock data and never automatically retry an owner-forwarded mutation after an ambiguous transport loss unless the owner can recover the same replay record.

Cancellation or deadline before the commit gate aborts. Cancellation after commit cannot undo state; retry must recover the committed result. Heartbeats, waits, and polling use `min(request timeout, gRPC deadline, server cap)` and do not hold database transactions while sleeping.

### Errors

Keep `vmond/src/api/grpc.rs::status_from` as the internal error taxonomy, then apply a fixture-backed legacy translation at the boundary. Encode standard gRPC status and, where expected, `modal.client.RPCStatus` in `grpc-status-details-bin`. Unknown rich-detail types are not invented. Preserve `vmon-code` only as harmless additional metadata.

Return `UNIMPLEMENTED` for unavailable capabilities; `INVALID_ARGUMENT` for malformed fields; `NOT_FOUND` without tenant disclosure; `FAILED_PRECONDITION`/`ABORTED` for state or fencing conflicts; `RESOURCE_EXHAUSTED` for explicit limits; `DATA_LOSS` for verified corruption; and `UNAVAILABLE` only for retryable infrastructure failures. Exact distinctions and messages are frozen by golden client tests.

### Streaming and backpressure

The scoped service has exactly 11 server streams: `AppGetLogs`, both container output streams, `DictContents`, both function data streams, `ImageJoinStreaming`, `SandboxGetLogs`, `SharedVolumeListFilesStream`, and both volume list streams.

Each stream reads bounded pages into a small bounded channel, awaits channel capacity, and stops its producer when the receiver is dropped. Durable sequence/cursor stateâ€”not a Tokio channel or timestampâ€”is the source of replay truth. A reconnect cursor is exclusive unless the selected profile proves otherwise. Streams pin one authorization/resource generation, honor deadlines and cancellation, and never follow a foreign owner after authorization changes without revalidation.

Unary uploads/downloads still obey the existing 64 MiB gRPC message ceiling. Larger content uses verified artifacts or scoped HTTP transfer URLs where the legacy response supports them. The control plane never accumulates unbounded logs, file trees, queue contents, or whole volume generations.

### Executable and secret payloads

Pickle/cloudpickle fields are executable data. Validate format, size, digest, tenant, serializer policy, and runtime ABI, but do not deserialize them in the daemon. Execute only inside the intended guest/worker trust boundary. Secret dictionaries and auth fields use zeroizing memory, encrypted tenant-bound storage where persistence is required, metadata-only list responses, and redacted audit/errors.

## State ownership

| Concern | Source of truth and implementation rule |
|---|---|
| Apps/functions/calls | `FunctionDomain` and its durable store own immutable revisions, activations, calls, inputs, results, and leases. Compatibility tables own only legacy IDs, drafts, aliases, bindings, app/call attribution, and cursors. |
| Sandboxes/snapshots | Engine registry and mesh ownership remain canonical. Compatibility state owns name/task projections, async-operation IDs, readiness/log cursors, and scoped session grants; it never duplicates VM lifecycle. |
| Artifacts/images/mount files | Canonical content-addressed storage verifies digests. Image tags and legacy blob/mount IDs are aliases to immutable content, not mutable copies. |
| Dicts | New tenant/environment-scoped transactional key/value domain with bounded values, inactivity expiry, atomic update/pop/clear, stable list order, and finite snapshot streaming. |
| Queues | New tenant/environment-scoped partitioned FIFO domain with durable sequences, TTL, retained entries for non-mutating iteration, atomic batch put/take, wait notifications, and mutation replay. |
| Environments | New policy/RBAC/admission domain. Billing/budget responses require a real fixed-point usage ledger and durable reservations; process-local metrics cannot masquerade as accounting. |
| Secrets | New generic encrypted environment-dictionary secret provider, separate from HTTP `CredentialService`; values never appear in list/get metadata. |
| Tasks/workers | New authenticated task/session records linked to canonical fenced input leases. Calls are not relabeled as tasks and orchestration worker heartbeats are not task membership. |
| Volumes | A canonical tenant-aware volume domain adds opaque IDs, immutable generations, metadata, quotas, transfer leases, path-safe file operations, and keyed attachment leases over existing encrypted archive/lock primitives. |
| Domains/endpoints/proxies/Flash | Require real DNS/certificate, serving endpoint, egress proxy/IP, or autoscaling control planes. Superficial mapping to sandbox ports or mesh nodes is forbidden; unsupported calls stay disabled. |

Local development may use SQLite, but mesh mode needs shared PostgreSQL-backed state or authoritative owner routing. Any local-only table that affects globally addressable IDs, queue/dict data, idempotency, or async operations is a correctness bug in cluster mode.

## Implementation sequence

1. **Freeze the wire contract.** Vendor/attribute the selected schema, trim the service allowlist to 212 methods, generate `modal::client`, pin the descriptor/hash, and capture supported Python/Go client fixtures. Assert 201 unary plus 11 server-streaming methods and zero extras.
2. **Mount a fail-closed shell.** Add the opt-in native route, exact path classification, credential translation/redaction, version gate, exhaustive method policy, legacy status/details encoding, message limits, and generic idempotency store. Every method initially returns intentional `UNIMPLEMENTED` rather than tonic's unknown-route response.
3. **Extract canonical application services.** Move reusable authorization, owner forwarding, app/call/sandbox/artifact/image/volume operations out of transport-only handlers without changing `vmon.v1` behavior.
4. **Ship direct and safe composite adapters.** Read-only lookups/lists, immutable artifact reads, call cancellation/results, sandbox lifecycle/snapshot/tunnel operations, and version-specific projections come first. Enable each method only after field-by-field golden fixtures pass.
5. **Add compatibility projections.** Implement tenant-scoped ID/name mappings, response replay, app/function bindings, log journals, async operation records, readiness/task projections, scoped grants, and recovery workers. Co-commit or recover every canonical/projection mutation.
6. **Add reusable missing domains.** Implement dict, queue, generic secret, environment policy, task/session, and volume-generation stores with limits, encryption, fencing, cluster semantics, and fault tests. Expose only methods whose full semantics are now truthful.
7. **Keep unsupported control planes closed.** Billing, DNS/certificates, serving endpoints, Flash, public IP proxy allocation, live volume commit/reload, command-router tokens, and any unimplemented worker plane remain `UNIMPLEMENTED` until their real domains exist.
8. **Run conformance and fault qualification.** Freeze the supported client-version matrix only after wire, restart, cancellation, retry, tenant isolation, cluster, and real-VM scenarios pass.

## Verification plan

- **Descriptor:** compare package/service path, all 212 method names, request/response full names, streaming flags, every used field number/oneof/enum/reserved range, and reject the 20 omitted upstream methods.
- **Stock clients:** run pinned official Python and Go clients against the opt-in endpoint for every enabled profile; preserve binary golden requests/responses and status details.
- **Authorization:** test UDS admin, valid/invalid token pairs, tenant/admin/worker/disabled policy, secret-header redaction, cross-tenant IDs/cursors/lists, and owner forwarding without credential leakage.
- **Idempotency:** inject connection loss before and after each mutation commit. Identical key/request returns the original outcome; changed request fails; destructive reads do not double-consume; restarts preserve replay.
- **Streaming:** reconnect at every cursor boundary, cancel slow consumers, enforce channel/page/message limits, prove no loss/duplication, and verify one pinned resource generation per stream.
- **State machines:** race create/get-or-create, publish/rollback, lease expiry/retry, queue take/clear/delete, snapshot/restore/terminate/wait, secret rotation, and volume publish/delete. Assert atomicity, fencing, recovery, and no fake success.
- **Security:** fuzz paths, IDs, enums, lengths, compressed/executable payloads, SSRF fields, symlinks, signed URLs/capabilities, stale generations, and foreign ownership. Verify secrets never reach logs/errors/audits.
- **Cluster:** restart owners, move sandbox ownership, partition shared stores, and exercise concurrent schedulers. No globally visible resource may depend on one node's memory or private SQLite.
- **Real runtime:** run the specific gated sandbox/function/image/volume paths on KVM/HVF only after protocol-only tests pass.

## Method-by-method plan

The four dispositions used below mean:

- **direct adapter:** one canonical operation plus field/status translation;
- **composite adapter:** multiple canonical operations and/or a durable compatibility projection;
- **new domain:** no truthful canonical capability exists yet; build an internal reusable domain before enabling the RPC;
- **UNIMPLEMENTED:** no safe implementation is presently available, or the required product/control-plane semantics are intentionally absent.

The ten drafts classify the end-state work as **13 direct adapters**, **60 composite adapters**, **104 methods requiring a new reusable domain**, and **35 methods that should remain `UNIMPLEMENTED`** with current capabilities. â€śNew domainâ€ť is a design target, not a claim of present support; those methods stay disabled until the named state machine, persistence, authorization, and tests exist.

## Release blockers and open questions

The descriptor and client source resolved the wire shapes, but they did not resolve every operational contract. Do not enable affected methods until fixtures or product decisions answer:

- the final supported client versions, exact rich error details, cursor inclusivity/expiry, log batch EOF/gap behavior, timestamp rounding, list caps/order, and heartbeat/ephemeral lease durations;
- a durable tenant/environment owner on canonical function revisions, calls, artifacts, and scheduler leases, including shared-store/owner routing in mesh mode;
- worker/task credential issuance, claims, rotation, revocation, audience, and generation fencing;
- deployment-name normalization and exact app deploy/publish/rollover/stop behavior across ephemeral, detached, and deployed states;
- cloudpickle/runtime compatibility policy and where argument binding/deserialization executes;
- real accounting, identity-directory, DNS/certificate, serving-endpoint, public IP proxy, Flash/autoscaling, and command-router control planes;
- canonical framed log journals, cumulative resource accounting, sandbox entrypoint stdin sessions, and readiness records;
- volume creation-mode lifetimes, V1/V2 path/copy semantics, quota values, transfer TTLs, guest freeze/dirty tracking, live commit, and reload.

The safe default for every unresolved item is deterministic `UNIMPLEMENTED`, not an approximation.
